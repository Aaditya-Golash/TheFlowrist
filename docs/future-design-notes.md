# Future design notes

These ideas came from the SSENSE/Aritzia design reference, but are not appropriate for the current server-rendered local pilot.

## Not added now

- Lenis smooth scrolling: requires a client-side runtime and extra dependency. Revisit only if the app gets a richer client layer.
- Next.js View Transitions: not relevant because this app is plain Node server-rendered HTML, not Next.js.
- React selection-card components: not relevant without React.
- Full-bleed cinematic video or high-art photography: useful later, but needs real brand assets and performance review.
- Physical QR recipient loop: a fulfillment and growth initiative, not a localhost UI readiness task.
- OpenRouter-generated card copy: would add AI/vendor surface area and needs consent, safety, and ops review first.
- Public recipient conversion offers or discounts: defer until payment, fulfillment, and support policies are real.
- Claimed churn metrics: keep out of product copy until supported by actual operating data.

## Later UI candidates

- A sharper editorial landing composition using real brand photography.
- Printed-card design specs for florist partners.
- A dedicated recipient referral flow after fulfillment is proven.
- More formal Concierge Vault onboarding once real payment tokenization exists.
